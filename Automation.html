What is Automated-Testing..?

To test the any application without any manual navigation that is called automation testing.

Automation Testing Using Selenium Webdriver....

Selenium Web Driver: using selenium web driver we can perform Automation Testing on Web Applications. Selenium web driver is open source tool. And we can customized the script easily. And many languages to support with selenium web driver. We can perform multiples script languages with selenium web driver. Like:

  • Java
  • C#
  • Python
  • Ruby
  • Perl

    • In selenium we have web elements, In this web elements we can perform action by using locators provided by selenium.

    Selenium provided 8 types of locators-

    1. Xpath
    2. CSS
    3. Class Name
    4. Link Text
    5. ID Name
    6. Partial link text
    7. Tag Name

    X path is also two types.

  • Relative x path: if x path is started with // then we can say it is relative x path, and relative x path we can create with 1 and more attributes like: combination of class name and id like that. We create dynamic x path approx. 13 types. We have around 13 types of methods we can use and create dynamic x path.
  • Absolute x path: if x path is started with / then we can say it is absolute x path.

    • What is TestNG in Automation-testing...?

    Test-NG is a next generation framework, it is inspire by J-unit, Test-Ng is provide some predefine classes, interfaces and methods, that is help to automate web applications or to perform action over the web elements. Test-Ng provides some annotations that will be help to perform one action’s to repeated way. And also achieve code reusability.

    There are some TestNG annotations like.....
    1. Before Suite
    2. Before Test
    3. Before Class
    4. Before Method
    5. Test
    6. Ater Method
    7. After Class
    8. After Test
    9. After Suite

    Using Test-Ng we can run parallel testing, cross browser testing and group by testing, but that was not possible in J-unit.

    Test-NG Report

    Test-Ng is providing reports that will be created automatically. In this reports we check which test case passed or which test cases is failed. Test-NG provide listeners, using this listeners we customized the reports.

    How many Types of assertions in Test-NG...?

    Test-NG framework provides assertions; In Test-NG we have 2 types of assertions....

  • Hard Assert:-

    If actual and expected result will not match then test case immediate fail. And fail reason is automatic impact in HTML reports. In Hard Assert have all the methods are statics.

  • Soft Assert (Verify):-

    If actual and expected result will not match then test cases will not fail till assert all () statement not executed. In soft assert all the methods are non-static. And same fail reason will be impact on html reports.

    • Wait in Selenium...

    Dynamic Wait: Selenium we have 3 types-

  • Implicit wait:-

    Implicit wait to use for wait for the loading time of a page. It will wait to maximum time, if in given between time page will be load then go to the further process, if page will not load then throw the time out exception.

  • Explicit wait:-

    Explicit to perform on particular conditions. Like- to be check visibility of element, element to be clickable, alert is present like that. In Explicit throw the multiple exceptions like- element not intractable, element is visible, alert is not present.

  • Fluent wait:-

    Fluent wait is used to define maximum time for the web driver to wait for a condition, using fluent wait to frequency with which we want to check a condition before throwing the exceptions. It checks for the web elements at regular intervals until the elements is found or timeout happens.

    • Selenium we perform the wait for loading time of the page, wait for the conditions and wait the condition in polling time interval.
